# ppt-template-fill 使用教程

两种用法：**方式一**是给 AI Agent（Claude Code / Codex 等）用的，你只管说话；**方式二**是自己跑命令行，适合调试或了解原理。日常用方式一即可。

---

## 0. 准备工作（一次性）

**1) 环境检查**

```powershell
python --version          # 需要 3.10+
pip show python-pptx      # 没有就: pip install python-pptx
```

截图功能（用于选版式和成品质检）需要本机装有 PowerPoint（Windows）或 LibreOffice。都没有也能用，只是 AI 看不到页面长相，纯靠文字契约工作。

**2) 安装 skill 到 Claude Code**

把仓库克隆（或复制）到用户技能目录：

```powershell
# Windows PowerShell
git clone https://github.com/xiongwenhao112/ppt-template-fill.git "$env:USERPROFILE\.claude\skills\ppt-template-fill"
```

```bash
# macOS / Linux
git clone https://github.com/xiongwenhao112/ppt-template-fill.git ~/.claude/skills/ppt-template-fill
```

其他 Agent（Codex 等）：放到对应技能目录，或直接让它读 `SKILL.md`。

**3) 准备一个模板**

任意 `.pptx` 都行——公司模板、学校课件模板、从模板网站下载的文件。模板里的每一页都会成为候选版式，所以**模板页数越全（封面/目录/章节页/几种正文页/结尾页），成品越丰富**。

## 1. 方式一：让 Agent 全自动做（推荐）

你只需要提供两样东西：**模板路径** + **想讲什么**。

### 提示词示例

最简单的说法：

> 用 `D:\模板\公司模板.pptx` 这个模板，帮我做一份《2026 上半年销售复盘》的 PPT，10 页左右，面向管理层。

给素材让它提炼内容：

> 用 `D:\模板\课件模板.pptx` 这个模板，把 `D:\讲义\第三章.docx` 的内容做成一份教学 PPT。

整体委托（不想被追问细节）：

> 用这个模板做一份 XX 主题的 PPT，内容你来定，不用问我，直接生成。

补充图片素材：

> 图片用 `D:\素材\` 里的几张架构图，放在合适的页面。

### Agent 会经历什么（你会看到的过程）

1. 解析模板 → 得到每页的可填槽位清单
2. 给模板逐页截图并查看 → 判断每页适合放什么内容
3. 规划章节结构 → 写出内容计划 plan.json
4. 校验计划（字数超预算、漏填槽位都会被拦下来修正）
5. 生成成品 → 再截图逐页自查（文字溢出、模板文案残留）
6. 给你成品 `.pptx` 的路径

### 怎么改稿

产物是**原生可编辑 PPTX**，两种改法：

- **让 Agent 改**：「第 3 页标题换成……」「第 5 页正文太挤，精简一半」「目录第 2 项和章节页对不上，统一一下」——它会改计划重新生成；
- **自己改**：直接用 PowerPoint 打开改，和普通 PPT 无异。

小改建议自己动手，结构性调整（增删页、换版式）交给 Agent。

## 2. 方式二：手动命令行操作

以下 `<skill>` 代表 skill 目录，`tpl.pptx` 代表你的模板。建议在一个工作目录里操作。

### 第 1 步：看模板全貌

```powershell
python <skill>\scripts\inspect_template.py tpl.pptx --compact
```

输出每页一行，例如：

```
p04 content  文本槽x2  版式'4_Title Slide'  核心必要性（六大痛点驱动） | 1. 解决「单人作坊」…
```

含义：第 4 页，角色猜测是正文页，有 2 个可填文本槽，后面是各槽的样本文字开头。

### 第 2 步：导出完整契约和截图

```powershell
python <skill>\scripts\inspect_template.py tpl.pptx --out manifest.json
python <skill>\scripts\snapshot.py tpl.pptx thumbs\
```

打开 `thumbs\slide-XX.png` 对照 `manifest.json` 看：每页哪些槽能填（`text_slots`）、字数预算（`max_chars`）、哪些是装饰不能动（`decorative: true`）、图片槽宽高比（`aspect`）。

### 第 3 步：写内容计划 plan.json

```json
{
  "template": "tpl.pptx",
  "output": "成品.pptx",
  "slides": [
    { "source": 1, "texts": { "2": "封面主标题", "3": "汇报人：张三\n2026.07" } },
    { "source": 4, "texts": { "5": "本页标题", "2": "第一段\n\n第二段\n\n第三段" } },
    { "source": 4, "texts": { "5": "另一页标题", "2": "正文页可以重复使用" } },
    { "source": 8 }
  ]
}
```

要点（完整规范见 `references/plan-format.md`）：

- `source` = 模板页码（1 开始），**同一页可以反复用**，正文页克隆多次是常态
- `texts` 的键 = manifest 里的槽位 `id`；表格单元格写 `"表格ID!r2c3"`
- 选了某页，它所有非装饰文本槽**都要填**，漏填的会残留模板演示文案
- 字数别超 `max_chars`；`\n` 表示分段
- 换图：`"images": { "22": "素材图.png" }`（路径相对 plan.json，自动按槽位比例裁切）
- 加演讲备注：`"notes": "这页讲 30 秒"`

### 第 4 步：先校验，再生成

```powershell
python <skill>\scripts\build_deck.py plan.json --check   # 只校验
python <skill>\scripts\build_deck.py plan.json           # 校验+生成
```

- `E:` 开头是错误（槽位不存在、图片找不到），必须修，否则拒绝生成
- `W:` 开头是警告（超字数、漏填），能生成但强烈建议处理

### 第 5 步：成品质检

```powershell
python <skill>\scripts\snapshot.py 成品.pptx check\
```

逐张看 `check\` 里的截图，重点：文字有没有溢出/压到装饰元素、有没有残留模板原文、图片有没有裁得难看。有问题改 plan.json 重跑第 4 步即可——生成是幂等的，随便重来。

## 3. manifest 速查表

| 字段 | 意思 | 用法 |
|------|------|------|
| `text_slots[].id` | 槽位 ID | plan.json `texts` 的键 |
| `sample` / `sample_len` | 模板演示文案及字数 | 新文案长度最可靠的参照 |
| `max_chars` / `max_lines` | 估算预算 | 超了会被 `--check` 警告 |
| `paragraphs` | 样本分段数 | 新文案分段数尽量一致 |
| `decorative: true` | 装饰槽（页码/序号/LOGO） | **不要填** |
| `layout_prompt` | 空占位符的用途提示 | 判断该填标题还是正文 |
| `image_slots[].aspect` | 图片槽宽高比 | 选图参考，差太多会被裁 |
| `table_slots[].cells` | 表格各格样本 | 按 `"ID!r行c列"` 逐格填 |

## 4. 常见问题

**Q：校验警告「未填，将残留模板文案」？**
把该槽位加进 `texts` 填上内容。确实想保留模板原文（如封面上的固定品牌行），就把原文照抄一遍填进去，警告消失且语义明确。

**Q：文字溢出/压到别的元素？**
最有效的办法是精简文案——引擎不会自动缩小字号。标题写短语别写句子；正文控制在 `max_chars` 内。

**Q：截图一直失败？**
确认装了 PowerPoint 且没有弹窗卡住；或者跳过截图步骤，纯靠 manifest 也能干活，只是少了视觉质检。

**Q：模板正文版式太少，10 页 PPT 全长一个样？**
这是模板本身的限制。要么换页数更全的模板，要么接受重复（靠内容分段撑起信息量），不要让 AI 硬造视觉。

**Q：能改配色/字体/布局吗？**
不能，这是特性：模板长什么样成品就长什么样，保证稳定。想换风格 = 换模板。

**Q：图表（Chart）/SmartArt 的数据能改吗？**
v1 不支持，会原样保留。规划时避开这类页，或接受其默认数据。

**Q：表格能加一行吗？**
不能，行列数固定，只能改格子里的文字。
